<h3>n-Tier Architecture</h3>
<p style= "padding-left: 10px;">
Is a design pattern that maps client-server model, breaking system into different pieces, which a tier only interacts with the tiers adjacent to it through a well-defined interface. Tiers can be upgraded or replaced at any time, just need to be the same interfaces as the old one
</p>
<h3>3-Tier Architecture</h3>
<p style= "padding-left: 10px;">
The 3-Tier architecture is basicaly Presetation tier -  The user interface. <br>
Application (Logic) tier - Modifies/delete sources and send the results. communicatino with the 2 others tiers. <br>
Data tier - The source of the data asssociated with the application
</p>
<h3>6-Tier Architecture</h3>
<p style= "padding-left: 10px;">
The application tier is often subdivided into two tiers. <br>
<h4 style= "padding-left: 10px;">Application</h4><p style= "padding-left: 10px;">
Business logic tier - Models the business objects associated with the application, all things associated with rules and process workflows, and how it can be manipulated. <br>
Data access tier - Responsible for accessing data, and passsing to the business logic tier. <br>
</p>
<h4 style= "padding-left: 10px;">Presentation</h4>
<p style= "padding-left: 10px;">Client tier - user interface components <br>
Presentation logic tier - scripts for generating web pages <br>
At last Web Tier </p>
