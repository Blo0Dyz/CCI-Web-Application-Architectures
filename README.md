# Web Application Architectures

<h3>Definition</h3>
<p style= "padding-left: 10px;">
A Web application is acessed by users over a network, using a browser as the client, and consists of a collection of client- and server-side scripts, HTML pages and other resources that may be spread across multiple servers 
</p>

<div class = "logicSide">
<h3>Client-Sever Model</h3>
<p style= "padding-left: 10px;">
A request/response protocol associated with any client-server architecture: <br>
1: request = Client -> Network -> Server <br>
2: response = Client <- Newtwork <- Server <br>
Client - Browser;
Network - HTTP;
Server - Webserver </p>

# Architectures

<div class = "Web10" ><h4>Web 1.0</h4>
<p style= "padding-left: 10px;">In Web 1.0 architecture have not much interactivity, and no separation of data, it only need to render HTML
<br>
Web Client (Browser) make 1:request -> Network -> Web Servers <---> Web Pages, Returning the 2:Response </p>
<h4>Web 2.0 and 3.0</h4>
<p style= "padding-left: 10px;"> in Web 2.0 and 3.0 architecture are better organized, with server-side functionality, makes the browser more capable
<br>
Web Client (Browser) make 1:request -> Network -> Web Servers <---> Script Service <---> Connector[Database] , Returning the 2:Response
</p>
